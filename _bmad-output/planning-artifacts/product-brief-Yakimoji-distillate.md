# Product Brief Distillate: Yakimoji

## Core Positioning

- Yakimoji 第一阶段不是“通用视频翻译工具”，而是“面向高频汉化创作者的频道配方工作台”。
- 最核心的产品资产不是转录、翻译或烤制能力本身，而是“频道配方”。
- `频道配方` 指围绕来源频道沉淀的默认规则集合，至少包含：来源识别键、语言对、字幕模板、输出平台偏好，以及可能的术语/风格偏好。
- Brief 中故意保留了高层体验主张，但没有展开到设计 token、字体、色值、组件规格级别；后续 PRD / UX spec 可继续吸收 `DESIGN.md` 中的风格线索。

## User and Workflow Signals

- 主用户是高频汉化型个人创作者，尤其是持续处理固定来源频道内容的一人内容工厂。
- 这类用户的核心痛点不是“不会翻译视频”，而是每次都要重复做相同配置决策。
- 用户成功感来自“导入后高命中率地带出默认规则”，不是“可以编辑很多东西”。
- 一阶段应强调“手动导入 + 高命中率复用”的工作流，而不是先做重型自动化编排。
- 交付物优先级：成品视频优先，字幕文件作为附带资产。

## MVP Workflow and Scope Signals

- MVP 主路径：导入链接/上传视频 -> 识别来源频道 -> 自动套用频道配方 -> 少量任务级覆盖 -> 转录/翻译/字幕/烤制 -> 交付成品视频 -> 仅在必要时人工确认。
- `少量任务级覆盖` 目前最明确的是切换字幕模板；PRD 阶段需要决定是否还包含语言风格、输出平台切换等有限覆盖项。
- 一阶段需要坚持“少打扰”原则：除陌生频道和低置信度片段外，不主动把用户拉回复杂确认流。
- 一阶段明确不做：自动订阅与批处理编排、复杂逐句编辑器、完整模板商店、智能遮挡检测、大团队协作后台。

## Rejected or Deferred Directions

- `单次任务快速汉化工具` 被保留为备选传播口径，但不作为第一阶段产品核心，因为会削弱“频道配方”差异化心智。
- `轻量来源订阅台` 被判断为第二阶段更合适的扩展方向，不适合第一阶段直接切入。
- `译制质量控制台`、`平台模板商店` 没有被选为第一阶段主线，原因是它们更像增强层，而不是切入点。
- `智能画面遮挡检测` 被明确延后，理由是成本高、复杂度高、不是 MVP 的首要价值来源。

## Experience and Style Signals

- 用户明确问过“样式风格这类实现细节是否进入 brief”，结论是：brief 只保留高层体验和风格主张，不展开到设计实现细节。
- 可写入后续 PRD/UX 的体验原则：高效、克制、少打扰、像创作工作台而不是复杂后台。
- `DESIGN.md` 当前呈现出明显的审美方向参考，可作为后续 UX 规格或视觉基调输入，但不应在 PRD 里直接转写为 token 级系统，除非那时已经决定要严格沿用。

## Competitive and Market Signals

- 市场已有较多 AI 视频字幕、配音、翻译、编辑工具，说明底层能力正在商品化。
- 机会点不在“证明 AI 能处理视频”，而在“降低高频工作流中的重复决策与启动成本”。
- `Why now` 的核心逻辑：平台与工具生态已开始教育创作者接受多语言和自动配音，用户对这类能力的心理门槛下降；新的产品差异更依赖工作流设计，而不是单点模型能力。
- PRD 阶段需要进一步明确首批替代对象：是替代创作者当前拼接使用的多工具工作流，还是替代某一类具体视频翻译/字幕工具。

## Open Questions for PRD

- 频道配方的最小字段集到底有哪些，哪些是必须字段，哪些是可选字段。
- 陌生频道首次出现时的 onboarding 应该多轻，是否允许“一次任务内创建并保存配方”。
- 来源识别在 MVP 中需要多稳定，若识别不出频道，系统应该如何降级。
- 低置信度片段确认的交互形式是什么，是否只给摘要/时间点而非完整编辑器。
- 输出平台偏好在一阶段只是记录导出目标，还是会影响字幕样式和安全区模板选择。
- “字幕模板”是纯视觉模板，还是会绑定平台规则和安全边界。
- 一阶段是否需要显式支持 API 触发任务，还是先把入口聚焦在手动导入与上传。

## Success Criteria Hints

- PRD 里应把成功指标拆成：命中率、任务启动步骤、用户手动配置次数、成品视频交付率、配方复用率。
- 还应考虑一个验证“资产性”的指标：单用户创建多个频道配方后，任务频率是否明显提升。
- 若需要商业化前置信号，可跟踪首批用户是否愿意长期把 Yakimoji 作为固定入口，而非只在偶发任务中使用。
